# 2026-06-10 Drag Upload v1

## 本轮目标

补齐首页上传简历的拖拽交互，让用户除了点击选择文件之外，也可以直接把 PDF / Word 简历拖到页面中上传。

## 本轮改动

1. 将原来的上传解析流程抽成统一入口，点击上传和拖拽上传走同一条逻辑。
2. 在首页“导入已有简历”区域增加可见的拖拽投放区。
3. 为拖拽状态增加高亮反馈，提升用户对可上传区域的感知。
4. 保持原有的 PDF / DOC / DOCX 解析与文本自动回填能力不变。

## 主要改动文件

- `app/page.tsx`

## 验证结果

- 首页已出现“拖拽简历到这里也可以”的提示区域
- `npm run build` 通过
- 上传解析接口冒烟测试通过

## 说明

这次迭代主要解决的是交互入口不完整的问题，不涉及后端解析能力变更。
