# 2026-06-10 Track Version v1

## 本轮目标

把项目从“通用 AI 简历助手”推进到“面向财经类高校本科生的赛道化简历工作台”，建立更像产品壁垒的能力。

## 为什么这轮重要

如果产品只提供：

- 上传简历
- AI 润色
- JD 匹配

那么任何通用 AI 工具都可以快速复制，差异很弱。

这次迭代的目标，是把核心价值转向：

- 更懂财经校招赛道
- 更懂同一份经历如何翻译成不同岗位语言
- 更懂哪些证据能支撑岗位版本，哪些地方只是空泛表述

## 本轮改动

1. 新增财经赛道知识底座，内置审计税务、财务分析、行业研究、银行管培、商业分析五类岗位打法。
2. 新增 `/api/track-version`，基于赛道知识和原始简历生成“岗位版本包”。
3. 首页新增目标赛道选择区，不再只有通用润色和 JD 匹配。
4. 输出区新增：
   - 赛道定位
   - 招聘方关注点
   - 现有证据如何使用
   - 可直接放进简历的赛道版摘要与改写要点
   - 风险与下一步建议
5. 支持把赛道版摘要一键插入到上方简历开头，形成可继续润色和匹配的版本。
6. 修正“原始事实”和“赛道版改写”前后没有明显区别的问题，要求赛道改写必须绑定原始事实清单，并在后端校验改写结果是否真正发生变化。

## 主要改动文件

- `lib/track-playbooks.ts`
- `app/api/track-version/route.ts`
- `app/page.tsx`

## 验证结果

- `npm run build` 通过
- `AI 润色` 接口可正常返回
- `JD 匹配` 接口可正常返回
- `赛道版本` 接口可正常返回结构化结果

## 这版体现出的产品竞争力

这次迭代后，项目不再只是“让 AI 帮你写简历”，而是开始体现出：

1. 面向财经类高校求职场景的本地化岗位知识
2. 同一份原始经历面向不同赛道的翻译能力
3. 从原始事实到岗位版本的完整工作流

## 说明

这版已经适合在 pre 里强调“核心竞争力”，因为产品的差异点开始从大模型能力，转移到赛道知识和工作流设计。
