# AI 简历助手产品 PRD

- 文档版本：V1
- 更新时间：2026-06-10
- 文档状态：Draft
- 样板场景：以上海财经大学为例，面向财经类高校本科毕业生
- 当前仓库基线：已具备 `AI 润色` 与 `JD 匹配` 两个核心 API，可作为下一阶段产品化的底座

## 1. 项目背景

财经类高校本科毕业生在校招和实习求职中，往往面临下面几类典型问题：

- 简历高度同质化，教育背景和经历标题相似，难以拉开差异。
- 很多学生不知道如何把“课程项目、商赛、学生组织、实习杂项工作”翻译成岗位需要的表达。
- 同一个学生会同时投银行、券商、基金、四大、互联网数据分析等多个方向，但简历版本常常只有一份。
- 大多数通用 AI 简历工具更擅长英文或泛互联网岗位，对中文财经类岗位的语言风格、关键词和风险边界理解不够。
- 学生往往在临近投递时才开始改简历，缺少“快速成稿 + 对岗迭代 + 多版本管理”的连续流程。

因此，这个项目的核心不是再做一个泛化 AI 工具，而是做一个更懂中国财经类毕业生求职场景的 AI 简历助手。

## 2. 主要解决的问题

### 2.1 学生不知道该写什么

- 不知道哪些经历值得写
- 不知道如何量化成果
- 不知道不同岗位看重什么

### 2.2 学生不知道该怎么改

- 看到 JD 后，只知道“不匹配”，不知道具体缺什么
- 不知道该补关键词，还是该改经历表述
- 不知道同一段经历能否改写成不同岗位版本

### 2.3 通用工具不够贴近财经校招

- 缺少银行、券商、基金、四大、财务分析、数据分析等赛道模板
- 缺少财经类高校常见经历的表达映射
- 缺少中文本地化简历风格和校招语义

## 3. 目标用户

### 核心用户

- 上海财经大学及同类财经类高校的大三、大四本科生
- 已有基础经历，但不会包装和匹配岗位的学生
- 同时投递多个方向，需要快速产出多个简历版本的学生

### 次级用户

- 跨申产品、运营、数据分析岗位的财经类学生
- 辅导员、就业指导老师、校内职业发展中心

## 4. 产品定位

一句话定位：

> 一个更懂中国财经类高校毕业生求职语境的 AI 简历助手，帮助学生把真实经历转化为贴岗、可信、可导出的多版本简历。

## 5. 产品目标

### 核心目标

- 让用户在 10 分钟内完成第一版可投递简历
- 让用户围绕不同岗位方向快速生成 2 到 3 个版本
- 让用户看到具体的 JD 差距，而不只是一个分数
- 保证输出尽量忠于原始事实，不鼓励虚构经历

### 非目标

- 第一版不做自动海投
- 第一版不做复杂社交或导师协作体系
- 第一版不覆盖所有高校与全部求职赛道

## 6. 解决思路

核心思路不是“让模型直接生成一份漂亮简历”，而是把过程拆成四层：

1. 先收集用户的真实经历和求职目标
2. 再把经历映射到财经岗位能力标签
3. 然后生成贴岗版本并做 JD 对比
4. 最后让用户基于反馈继续微调和导出

## 7. 产品逻辑总览


## 8. 产品设计

### 8.1 信息输入层

目标是让学生“容易开始”，而不是一上来就被一大堆字段吓退。

建议输入项：

- 学校、专业、毕业时间
- 目标岗位赛道
- 实习经历
- 项目经历
- 课程项目 / 商赛 / 论文 / 社团
- 技能与证书
- 目标 JD

设计原则：

- 先问目标，再问经历
- 先收事实，再做润色
- 允许“只粘贴旧简历”快速启动
- 允许“结构化补录”提升质量

### 8.2 认知层

这是本项目区别于通用 AI 工具的关键。

需要内置的认知模块：

- 财经岗位词典：银行、券商、基金、四大、财务、数据分析
- 财经能力标签：研究、估值、建模、风控、沟通、数据处理、报表分析、行业洞察
- 校招经历翻译器：把课程项目、竞赛、社团、实训转换成岗位语言
- 风险约束器：禁止模型凭空增加实习、项目、技能和量化结果

### 8.3 生成层

生成层至少支持四类输出：

- 个人总结
- 经历要点改写
- 岗位定制版本
- JD 差距补充建议

生成原则：

- 优先保真，不优先华丽
- 优先贴岗位，不优先通用万能表述
- 优先给出“可替换文案”，不只给抽象建议

### 8.4 反馈层

反馈层不能只给一个分数，否则学生不知道下一步该做什么。

建议输出：

- 总体匹配度
- 已命中关键词
- 缺失关键词
- 需要改动的简历区块
- 一键改写建议
- 风险提示，比如“这条成果缺少证据支撑”

## 9. 功能模块设计

| 模块 | 功能说明 | 优先级 |
| --- | --- | --- |
| 新手引导 | 选择学校、专业、目标岗位赛道，明确生成目标 | P0 |
| 简历录入 | 支持粘贴旧简历，也支持结构化录入经历 | P0 |
| AI 润色 | 对个人总结、项目要点、实习要点进行改写 | P0 |
| JD 匹配 | 对照目标岗位输出分数、关键词和修改建议 | P0 |
| 岗位版本生成 | 同一份经历生成银行版、券商版、四大版等 | P1 |
| 多版本管理 | 保存多个岗位版本，方便比较和导出 | P1 |
| 上传解析 | 上传 PDF/Word 提取内容进入编辑态 | P2 |
| 模板导出 | 多模板 PDF 导出 | P2 |

## 10. 功能演示

### 演示 1：上海财经大学会计学本科，投四大会计审计岗

用户输入：

- 学校：上海财经大学
- 专业：会计学
- 经历：事务所审计实习、财务分析课程项目、校学生会经历
- 目标岗位：四大审计 / 咨询

系统输出：

- 将“整理底稿、辅助盘点”改写为更专业的审计表述
- 提示补充“报表分析、数据核对、Excel 处理效率、跨团队沟通”关键词
- 生成一版更适合四大的简历摘要

### 演示 2：上海财经大学金融学本科，投券商研究所实习

用户输入：

- 学校：上海财经大学
- 专业：金融学
- 经历：行业研究课程、券商实习、商赛路演
- 目标岗位：行业研究 / 行研助理

系统输出：

- 将“写过行业报告”改写为“完成行业资料收集、公司对比和观点整理”
- 提示补充“估值、财报、行业跟踪、Wind/Excel/Python”相关关键词
- 给出更贴研究岗的个人总结和项目经历写法

### 演示 3：财经类学生跨申数据分析岗

用户输入：

- 学校：财经类高校
- 专业：统计 / 金融工程 / 信息管理
- 经历：数据课程项目、竞赛、实习分析工作
- 目标岗位：数据分析

系统输出：

- 把“做过数据清洗”改写为更强调业务分析与结果的表述
- 对照 JD 提示补充 SQL、BI、指标体系、可视化等关键词
- 生成更偏互联网风格的数据分析版本

## 11. 当前仓库与产品规划的衔接

### 已有基础

- 首页已经具备简历全文粘贴与 JD 粘贴入口
- `/api/polish` 已支持 AI 润色
- `/api/match` 已支持 JD 匹配评分与建议
- 仓库中已经有结构化表单与预览相关组件，可继续复用

### 下一步最值得做的事

1. 把仓库内已有结构化组件重新接入首页流程
2. 增加“目标岗位赛道”选择
3. 增加“同一经历多版本改写”
4. 为上传解析预留统一 schema

## 12. 价值评估

### 对学生的价值

- 降低简历起步门槛
- 减少盲目改简历的时间
- 让同一份经历更贴近不同岗位
- 提高简历表达的可信度与针对性

### 对学校或就业指导场景的价值

- 可作为就业指导工具
- 可沉淀财经类岗位和优秀表达案例
- 可扩展成学院内部就业支持产品

### 对项目本身的价值

- 比通用 AI 简历工具更容易做出差异化
- 更适合从 MVP 逐步扩成“校招求职助手”
- 易于在后续拓展到上传解析、模板市场、岗位知识库

## 13. 主要风险

### 风险 1：模型幻觉导致简历失真

表现：

- 凭空补数据
- 夸大成果
- 编造技能或实习内容

应对：

- 严格要求只基于用户输入改写
- 所有润色结果必须可回溯到原始事实
- 对“量化表达”增加人工确认提醒

### 风险 2：用户隐私和简历敏感信息泄露

表现：

- 简历含手机号、邮箱、身份证、实习公司敏感信息

应对：

- 优先走最小化存储
- 明确提示用户数据用途
- 后续如果接数据库，要补充脱敏和权限设计

### 风险 3：过度偏向名校标签

表现：

- 工具可能错误强化“学校比经历更重要”的导向

应对：

- 把产品重点放在经历表达和岗位匹配，而不是学校排序
- 在文案和建议中减少标签化倾向

### 风险 4：国内职位数据抓取存在合规和稳定性风险

表现：

- 岗位来源接口变化
- 平台限制抓取

应对：

- 第一版不依赖大规模实时抓取
- 先从用户手动粘贴 JD 开始
- 后续再建设可维护的岗位语料库

### 风险 5：第一版做得太大

表现：

- 一次性做编辑器、解析、导出、模板、知识库、自动投递

应对：

- 第一版只做“录入/粘贴 -> 改写 -> 匹配 -> 版本生成”
- 把自动投递和重型解析延后

## 14. 里程碑建议

### Phase 1：产品化 MVP

- 完成结构化录入
- 增加目标岗位赛道
- 增强润色结果的可替换性
- 强化 JD 匹配反馈

### Phase 2：财经场景强化

- 建立财经岗位词典
- 建立中文校招经历表达模板
- 支持一份简历生成多个岗位版本

### Phase 3：完整工具链

- 上传 PDF / Word 解析
- 多模板导出
- 多份简历管理
- 学校 / 学院级知识库沉淀

## 15. 最终判断

这个项目有明确价值，关键不在于“AI 是否足够强”，而在于是否真正理解中国财经类高校毕业生的求职场景。

如果方向收敛为：

- 财经类学生
- 真实经历改写
- 贴岗版本生成
- JD 差距反馈

那么它比做一个泛泛的“AI 简历生成器”更容易形成产品辨识度，也更适合在当前仓库基础上继续迭代。
