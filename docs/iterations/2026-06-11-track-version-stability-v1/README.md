# 2026-06-11 Track Version Stability v1

## 本轮目标

修复“财务分析版本一直生成不了”的问题，把赛道版本生成功能调整成更适合 pre 演示的稳定版本。

## 问题现象

- 财务分析赛道经常长时间等待后失败
- 即使不报错，也可能回退成过于机械的模板句
- 一旦模型结构化输出不稳定，用户几乎感受不到可用结果

## 解决思路

1. 缩小模型需要承担的输出范围，让赛道版本接口优先保证“能稳定返回结果”。
2. 增加本地保底生成逻辑，即使外部智能服务不稳定，也能产出可演示的赛道版本内容。
3. 把保底稿从简单模板升级为“按事实类型 + 目标赛道”重写，确保原始事实和赛道版改写有明显差异。

## 本轮改动

1. 为大模型调用增加超时控制，避免赛道版本接口长时间卡住。
2. 调整 `/api/track-version` 的生成策略，优先请求更轻量的结构化结果。
3. 新增本地赛道化保底生成逻辑：
   - 自动识别教育背景、实习经历、项目经历、校园经历
   - 按审计税务、财务分析、行业研究、银行管培、商业分析五类赛道分别改写
   - 自动结合 JD 中出现的重点关键词生成定位摘要和改写内容
4. 优化财务分析赛道的保底文案，避免只是在原句前面机械加前缀。

## 主要改动文件

- `app/api/track-version/route.ts`
- `lib/llm.ts`

## 验证结果

- `npm run lint` 通过
- `npm run build` 通过
- 本地回归 `finance_analysis` 请求返回 `200`
- 本地回归 `audit_tax` 请求返回 `200`
- 财务分析赛道现在可以稳定返回“定位摘要 + 原始事实 + 赛道版改写 + 风险与建议”

## 结果

这次迭代后，财务分析版本不再出现“点了很久生成不了”的体验问题；即使模型输出不稳定，页面也能返回一版更像产品结果、可用于演示和继续修改的赛道稿。
