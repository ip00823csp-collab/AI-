# 2026-06-10 Demo Stability v1

## 本轮目标

优先解决 pre 演示场景下的稳定性问题，避免上传简历和 JD 后频繁出现“这次生成结果不够稳定，请再试一次”。

## 主要问题

1. 默认模型 `glm-4.7-flash` 在当前时段容易触发限流，影响现场演示连续性。
2. 结构化输出完全依赖模型一次性返回标准 JSON，容错空间过小。
3. 当模型返回格式轻微漂移时，后端会直接判定失败，导致前端体验不稳定。

## 本轮改动

1. 将默认模型切换为更适合演示稳定性的 `glm-4-flash-250414`。
2. 新增结构化输出专用调用流程，支持模型重试。
3. 增强 JSON 提取与解析容错，支持从更复杂的返回文本中提取合法 JSON。
4. 在接口层增加返回结构校验，避免模型返回半成品数据直接进入前端。
5. 保留 `GLM_MODEL` 和 `GLM_MODEL_FALLBACKS` 配置能力，方便后续切换模型。

## 主要改动文件

- `lib/llm.ts`
- `app/api/polish/route.ts`
- `app/api/match/route.ts`
- `.env.example`
- `README.md`

## 验证结果

- `npm run build` 通过
- `AI 润色` 接口可正常返回结构化结果
- `JD 匹配` 接口可正常返回结构化结果

## 说明

这次迭代优先保障“能稳定演示”，并不代表模型选择已经是最终长期方案。后续仍可以根据成本、速度和效果继续调整。
