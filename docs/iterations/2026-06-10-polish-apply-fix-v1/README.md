# 2026-06-10 Polish Apply Fix v1

## 本轮目标

修复“生成完表达建议后，点击用建议稿替换上方简历内容，看起来没有生效”的问题。

## 问题表现

用户在输出区点击“用建议稿替换上方简历内容”后，页面没有给出足够明确的反馈，也不会自动带用户回到上方简历输入框，因此容易误判为按钮无效。

## 本轮改动

1. 为简历输入框增加引用，替换完成后自动聚焦并滚动回输入区域。
2. 点击“用建议稿替换上方简历内容”后，立即显示明确的成功提示。
3. 用户继续手动编辑简历时，成功提示会自动清除，避免误导。
4. 保持原有 AI 润色与 JD 匹配逻辑不变，只修正交互反馈链路。

## 主要改动文件

- `app/page.tsx`

## 验证结果

- `npm run build` 通过
- 按钮点击后会触发聚焦、滚动和成功提示

## 说明

这次迭代主要解决的是“已生效但用户感知不到”的交互问题，重点是让替换动作可见、可确认。
